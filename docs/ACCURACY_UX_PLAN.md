# 高精度化 & UI 改善計画 — v1.1 に向けて

作成日: 2026-07-10
前提: 精度第1〜2波(文脈プロンプト/信頼度/正規化/文脈翻訳/用語強制/リスコア/単語再生)と
製品化 P0(テスト・CI・堅牢性・セットアップ・法務)は実装済み。

この計画は「**認識・翻訳をさらに正確に**」と「**試合中・復習中の操作性を上げる**」の 2 軸。
各項目に効果/コスト/検証方法を明記し、2 波に分けて実装する。

---

## 第 1 波(効果大・リスク小 — 先にやる)

### 精度

#### P1. 定型句の即時翻訳キャッシュ
- **何**: 完全一致 or 正規化一致した既訳(glossary 定型句 + 過去に確定した同一文)を
  LLM を呼ばずに即時返す 2 層キャッシュ(メモリ + history 参照)。
- **効果**: 頻出コール("nice"/"rotate B"/"reloading")の日本語が **0ms** で出る。
  LLM 負荷が減り、混雑時のサジェスト遅延も改善。訳の揺れも消える(同じ文は常に同じ訳)。
- **実装**: translate.py に `TranslationCache`(LRU 512 + glossary 完全一致表)。
  キャッシュヒット率をログに出す。
- **検証**: 同一文 2 回目が <10ms / ヒット率がテストで確認できること。

#### P2. 音声前処理(認識の底上げ)
- **何**: セグメントを Whisper に渡す前に (a) 80Hz ハイパス(ゲーム低音・ハム除去)、
  (b) ピーク正規化(小声の味方を適正レベルに)。numpy/scipy 級の軽量処理。
- **効果**: 小声・こもり声の認識改善。VC は音量差が激しいので効きやすい。
- **実装**: pipeline の STT 直前に `preprocess.py`(biquad ハイパス + normalize)。
  config `stt.preprocess: true` でオンオフ。
- **検証**: テスト音声を -20dB に落としたクリップで前処理有無の認識率を比較。

#### P3. ライブ中の低信頼行の自動リトライ
- **何**: 低信頼(≈)だった行を、GPU がアイドルになった瞬間に beam5 で再認識し、
  ライブ表示と履歴を更新(レビュー時リスコアの「ライブ版」)。
- **効果**: 試合中でも数秒後に ≈ が正しい文に置き替わる。
- **実装**: pipeline に低優先度キュー。seg_q が空 & 直近 2 秒発話なしのときだけ処理。
  既存の refine_utterance / refined イベントを流用。
- **検証**: 自己テストで低信頼行が refined イベントで置換されること。

#### P4. 動的 hotwords(直近の試合語彙を注入)
- **何**: そのセッションで確定した固有語(エージェント名・頻出語)を hotwords に追記し、
  以降の認識をバイアス(「一度聞き取れた名前は以後間違えない」)。
- **実装**: pipeline が確定行から `frequent_terms` 相当を回し、Transcriber.hotwords を更新。
- **検証**: 同一名詞の 2 回目以降の認識が安定することをログで確認。

### UI/操作性

#### U1. グローバルホットキー(ゲーム中に Alt+Tab 不要へ)
- **何**: ゲームがフォアグラウンドでも効くホットキー:
  `Ctrl+Alt+T` 翻訳の開始/停止 / `Ctrl+Alt+S` **直前の発言を★保存** /
  `Ctrl+Alt+O` オーバーレイ表示切替。
- **効果**: 「今のフレーズ覚えたい!」を試合中に 1 キーで保存できる — 学習アプリとして体験が段違い。
- **実装**: Win32 `RegisterHotKey` を専用スレッドで(依存追加なし)。config でキー変更可。
  保存時はオーバーレイの該当行に ★ を一瞬表示してフィードバック。
- **検証**: 別アプリがフォーカス中でも発火すること(自己テスト+手動)。

#### U2. オーバーレイの位置ドラッグ調整 + 設定の即時反映
- **何**: 設定画面に「位置を調整」ボタン → オーバーレイが一時的にクリック可能になり
  ドラッグで移動、確定で y_offset/x_offset を書き戻し。opacity 等のスライダーは
  稼働中のオーバーレイに即時反映。
- **効果**: 今は yaml の数値を想像で調整 → 見たまま合わせられる。
- **実装**: overlay に adjust モード(click-through 解除+ドラッグハンドラ)。
  bridge.set_setting からオーバーレイへ live 反映フック。

#### U3. トレイ常駐 + ウィンドウ状態記憶
- **何**: 閉じるボタンで「トレイに最小化(翻訳は継続)」を選択可。位置・サイズ・
  最終タブを data/ui-state.json に記憶して次回復元。
- **実装**: pywebview の hidden + Win32 トレイアイコン(pystray は avoid — ctypes Shell_NotifyIcon
  最小実装 or 単純に「最小化で隠す+ホットキーで復帰」)。まず後者の簡易版で価値検証。

#### U4. ライブ画面のスクロール改善
- **何**: ユーザーが上へスクロールしたら自動追従を止め、「↓ 最新へ」チップを表示。
  現在は毎行強制で最下部へ飛ぶため見返せない。
- **実装**: app.js で scrollTop 監視、追従フラグ。

#### U5. 復習リマインダ
- **何**: 起動時に due 件数をトースト表示(「復習が 12 件たまっています → 開く」)。
- **実装**: boot 時に due_count>0 でトースト + クリックで復習タブへ。

## 第 2 波(価値は高いがコスト/リスク中)

| # | 項目 | 概要 |
|---|---|---|
| P5 | 部分字幕(ストリーミング風) | 2 秒超の長発話を仮確定チャンクで先出しし、確定時に置換。長文での「無音の 6 秒」を解消 |
| P6 | ピッチ保持スロー再生 | 0.5×/0.75× を WSOLA でピッチ維持(現在は低音化)。リスニング練習の質向上 |
| P7 | 翻訳スタイル設定 | 「ですます/常体/ゲーマー口調」を設定から選択(システムプロンプト切替) |
| U6 | 本格トレイ(Shell_NotifyIcon) | 常駐メニュー(開始/停止/表示/終了) |
| U7 | 統計ミニダッシュボード | 週間: 記録行数・★保存・復習正答率の推移(ライブラリ上部に小さく) |
| U8 | オンボーディングツアー | 初回のみ各タブの 1 行ガイド(セットアップ完了後) |

## 実施順序

```
第1波: P1 → P2 → U1 → U2 → P3 → U4 → U5 → P4 → U3(簡易)
        └ 精度の即効 2 つ → 体験が跳ねるホットキー/位置調整 → 残り
第2波: 実戦フィードバックを見て優先度を再判定
```

各項目とも: 実装 → pytest 追加 → プレビュー/自己テスト検証 → コミット(CI green)→
第1波完了時に exe 再ビルド + CHANGELOG 1.1.0 起票。

## 検証の指標

- 精度: テストクリップ一式(通常/− 20dB/早口)の WER 相当の目視比較表を docs に記録
- 応答: 定型句の翻訳表示までの時間(目標 <50ms)、通常文は現状維持(~0.5s)
- 操作: 「試合中に Alt+Tab した回数」が実戦でゼロになること(U1 の成否基準)
