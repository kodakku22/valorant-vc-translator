# VC Translator 製品化計画書 — v1.0 リリースまでの道筋

作成日: 2026-07-04
現状: 機能完成・2 回のレビューサイクル済み(計 18 件修正)・個人利用では実用レベル
目標: **「他人の PC に配って、説明なしで使い始められ、壊れない」= 製品版 v1.0**

---

## 0. 現在地の評価

| 領域 | 現状 | 製品版とのギャップ |
|---|---|---|
| コア機能(翻訳・履歴・復習) | 動作検証済み。翻訳 0.5s / STT 0.3s 実測 | ほぼなし |
| UI(Editorial Ink) | 5 画面実装済み | 初回体験・エラー時の案内が弱い |
| 配布 | PyInstaller フォルダ 4.3GB + ショートカット手動作成 | インストーラなし・未署名・巨大 |
| セットアップ | VB-Cable / ルーティングは README 頼み | **最大の脱落ポイント**(手動 4 手順) |
| テスト | 検証スクリプトが scratchpad に散在 | リポジトリ内に自動テストなし・CI なし |
| 耐障害性 | 起動時の異常系は対応済み | **稼働中**の異常(デバイス抜け・Ollama 落ち)が未対応 |
| 法務・プライバシー | README に記載・OFL 同梱済み | アプリ内での同意取得なし・NOTICE 不足 |

結論: 機能ではなく **「配布・初回体験・稼働中の堅牢性・品質保証の仕組み」** が製品化の本丸。

## 1. フェーズ構成(P0 → P2)

### 進捗(2026-07 更新)

- ✅ **P0-1 テスト基盤+CI** — tests/ + GitHub Actions(compile / pytest / ruff / web構文)
- ✅ **P0-3 稼働中の堅牢性** — 入力自動再接続 / Ollama 稼働中復帰 / crash ログ
- ✅ **P0-2 セットアップ**(検査画面+ループバック音声テスト)/ **P0-5 同意・法務**(初回同意 / NOTICES / 免責 / バージョン)
- ⬜ **P0-4 配布**(Inno Setup インストーラ / サイズ削減) — 残タスク

現状は「機能・品質・テスト・堅牢性・法務」が揃い、**開発者ビルド(build_exe.bat)で配れる状態**。
一般配布用の**インストーラ化とサイズ削減が v1.0 リリース前の最後の関門**。

### P0 — v1.0 に必須(これがないと「製品」と呼べない)

#### P0-1: テスト基盤と CI(最初にやる — 以降の全変更の安全網)
- scratchpad の検証スクリプトを `tests/` に移植し pytest 化
  - `test_history.py`(スキーマ移行・SRS・聞き逃し・検索)
  - `test_config.py`(プロファイルマージ・設定書き込みスコープ・コメント保持)
  - `test_vad.py`(合成波形でのセグメンテーション・強制区切り)
  - `test_stt_filters.py`(幻聴フィルタ)/ `test_suggest_parse.py`(`||` パース)
  - `test_bridge.py`(モック音声でのライブフロー。Whisper は tiny、Ollama はモック)
- GitHub Actions: push ごとに pytest(CPU・tiny モデル)+ `node --check` + ruff lint
- **完了基準**: main への push で全テストが自動実行され green

#### P0-2: セットアップウィザード(最大の脱落要因を潰す)
- 初回起動時にアプリ内ウィザードを表示:
  1. VB-Cable 検出(なければ公式 DL リンク+インストール後の再起動案内)
  2. Valorant の VC 出力設定手順(スクリーンショット付き・アプリ内表示)
  3. 「このデバイスを聴く」設定手順+**ループバック音声テスト**(テスト音を CABLE に流し、取り込めたら ✓)
  4. Ollama 検出(なければ DL リンク)+翻訳モデルの pull 進捗表示
  5. Whisper モデルの DL 進捗表示(現状は無言で数分待たせている)
- 各ステップに「あとで」を用意し、未完了項目はライブ画面の状態バーに警告表示
- **完了基準**: まっさらな Windows 機で、README を読まずに初回翻訳成功まで到達できる

#### P0-3: 稼働中の堅牢性
- **デバイス喪失対応**: 試合中に CABLE が消えた/既定デバイス変更 → 状態バー `INPUT: LOST` +自動再接続リトライ(現状: capture スレッドが死んで無言)
- **Ollama 切断の自動復帰**: 稼働中に落ちたら再接続を試み、復帰したら翻訳再開(現状: 一度失敗すると英語のみで固定)
- **スリープ復帰**: レジューム後のストリーム再初期化
- **オーバーレイの別プロセス化**: 現在の「別スレッド tkinter」(公式サポート外)を小さな独立プロセス(stdin で字幕受信)に置換。クラッシュしても本体無傷・自動再起動
- クラッシュハンドラ: 未捕捉例外 → data/crash-*.log 保存+再起動ダイアログ
- **完了基準**: 「稼働中に VB-Cable 無効化 → 有効化」「Ollama kill → 再起動」「スリープ → 復帰」の 3 シナリオで、アプリ再起動なしに翻訳が復帰する

#### P0-4: 配布パッケージ
- **インストーラ化**(Inno Setup): インストール先選択・ショートカット・アンインストーラ・(データは %LOCALAPPDATA% へ移動し、アンインストール時に「学習データを残す」選択肢)
- **サイズ削減**: 現 4.3GB → 目標 2.5GB 以下
  - torch から不要コンポーネント除外(torchaudio 依存の見直し=silero-vad を onnxruntime 直接ロードに置換すれば torch 自体を排除可能。ctranslate2 用 cuDNN/cuBLAS DLL は個別同梱)
  - PyInstaller excludes の精査(tests / examples / 未使用 CUDA カーネル)
- **バージョニング**: semver 開始(v1.0.0)、`__version__` を UI(設定画面下部)と exe メタデータに表示、CHANGELOG.md 開始
- GitHub Releases にインストーラを添付、リリース CI(タグ push でビルド)
- SmartScreen: 署名は個人開発ではコスト高のため v1.0 では「詳細情報→実行」の案内を初回ページに明記(署名は P2)
- **完了基準**: 別 PC で installer.exe 実行 → ウィザード → 翻訳成功、まで一本道

#### P0-5: プライバシー・法務の仕上げ
- 初回起動時の同意画面: 「味方の音声を**ローカルにのみ**録音・保存する」ことの明示と、配信に映さない注意(チェックボックス)
- THIRD-PARTY-NOTICES.txt 生成(pip-licenses で全依存の許諾を同梱。faster-whisper/MIT、torch/BSD 等)
- README に Riot Games 非公式ツールである旨の免責を明記
- **完了基準**: 同梱物のライセンス由来がすべて追跡可能、録音同意がアプリ内で取れている

### P1 — v1.1(製品の完成度を上げる)

- **データ管理**: 設定画面に「ストレージ使用量表示+古い音声の自動削除(30日/上限GB)」、履歴の CSV / **Anki エクスポート**、DB バックアップ・復元
- **UX 磨き**: ウィンドウ位置・サイズ・最終タブの記憶 / システムトレイ最小化(試合中はトレイ常駐)/ ショートカットキー(開始停止のグローバルホットキー)/ オーバーレイ位置のドラッグ調整モード
- **稼働中テレメトリ表示**: VRAM 使用量と「モデルが CPU フォールバックした」ことの明示(現状ログのみ→性能急落の原因が見えない)
- **学習機能の質**: カードの重複統合(同一フレーズ)/ マスク語の品質改善(頻出コール用語を優先マスク)/ SRS パラメータの見直し(easiness 導入)/ 復習リマインダ(起動時に due 件数トースト)
- **音声品質**: 再生の時間伸縮を「ピッチ維持」(WSOLA/phase vocoder or `audiotsm`)に — 現在は単純リサンプルでピッチが下がる
- **完了基準**: 1 週間の常用でストレージ・ウィンドウ管理・復習動線にストレスがないこと

### P2 — v1.2+(あれば嬉しい)

- コード署名証明書(SmartScreen 警告解消)
- 自動アップデート(起動時に GitHub Releases をチェック → ダウンロード案内)
- 多言語 VC 対応(言語自動判定: 英語以外の VC も翻訳)/ UI 英語化
- Discord 通話モード(ルーティングプリセット切替)
- 統計ダッシュボード(週間の聞き取れた率・保存語彙数の推移)
- オーバーレイの webview 化(角丸・真の透過などデザイン完全再現)

## 4. 実施順序とマイルストーン

```
M1: P0-1 テスト基盤+CI            ← 全変更の安全網を最初に
M2: P0-3 稼働中の堅牢性            ← 挙動変更を伴うのでテスト直後に
M3: P0-2 セットアップウィザード     ← UI 追加(バックエンド安定後)
M4: P0-4 配布(インストーラ/サイズ) ← 中身確定後にパッケージ
M5: P0-5 法務+QA マトリクス+リリース
```

依存関係: M1 → M2 → (M3 と M4 は並行可) → M5。
サイズ削減の torch 排除(P0-4)は影響が大きいため、M1 のテストが揃ってから着手する。

## 5. リスクと逃げ道

| リスク | 逃げ道 |
|---|---|
| torch 排除(silero→onnxruntime 直)が難航 | v1.0 は 4.3GB のまま出し、サイズ削減は v1.1 に送る |
| オーバーレイ別プロセス化で遅延増 | 現行スレッド方式を fallback として残す設定フラグ |
| 署名なしの SmartScreen 離脱 | 初回ページに回避手順を明記。DL 数が伸びたら署名(P2)前倒し |
| CPU 環境での実用性(STT 遅い) | ウィザードで GPU なし検出時に base/medium を既定提案 |
