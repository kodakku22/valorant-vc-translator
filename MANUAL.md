# Valorant VC リアルタイム翻訳 — 操作マニュアル

英語のボイスチャットを、完全ローカルで **英日併記の字幕** にして画面に表示するツールの
使い方マニュアルです。初めての人でもこの順番どおりに進めれば動きます。

- 対象環境: Windows 11 / RTX 5070 Ti(GPU 翻訳)
- 遅延の目安: 発話終わりから **日本語表示まで約 0.6〜0.9 秒**(実測)
- 音声は一切外部に送信されません(すべて PC 内で処理)

## 1. これは何をするツールか

Valorant の味方の英語 VC を、**仮想オーディオケーブル**で分岐させて取り込み、
音声認識(faster-whisper)→ 翻訳(Ollama / gemma4)を通して、
画面下の小窓に **英語原文と日本語訳を2段** で表示します。

```
味方の英語VC → VB-Cable → 認識 → 翻訳 → 字幕(英語が先に出て、日本語が追って入る)
       ↘ 自分のヘッドホンにも分岐(声はちゃんと聞こえる)
```

ゲーム音(銃声・足音)はこれまで通り自分のヘッドホンから鳴ります。
翻訳パイプラインに入るのは VC の声だけなので、ノイズが混ざらず認識精度が高いのが特長です。

## 2. 初回セットアップ(1回だけ)

> ソフト側の環境構築(Python 仮想環境・GPU 版 PyTorch・音声認識モデル・翻訳モデル)は
> **すでに完了しています**。ここでやるのは音声のルーティング設定だけです。

### ステップ A: VB-Cable が入っているか確認

すでにインストール・再起動済みです。念のため確認するには、コマンドプロンプトで:

```bat
cd "C:\claude code\valorant-vc-translator"
run.bat --list-devices
```

一覧に **`CABLE Output (VB-Audio Virtual Cable)`** が出ていれば OK です。

### ステップ B: Valorant の VC 出力を仮想ケーブルへ

1. Valorant を起動 → 右上の歯車 → **設定**
2. **「オーディオ」タブ → 「ボイスチャット」**
3. **「出力デバイス」** を **「CABLE Input (VB-Audio Virtual Cable)」** に変更

> ⚠️ 変えるのは **ボイスチャットの「出力デバイス」だけ** です。
> - マイク(入力デバイス)は **HyperX QuadCast S のまま** → 自分の声はこれまで通り味方に届きます
> - ゲーム音の出力(オーディオタブ上部)も **触らない** → 銃声・足音は普段のデバイスから鳴ります

### ステップ C: 自分にも VC が聞こえるようにする

このままだと味方の声が仮想ケーブルに吸われて自分に聞こえなくなるので、分岐させます。

1. タスクバーのスピーカーアイコンを右クリック → **「サウンドの設定」**
2. 下の方の **「サウンドの詳細設定」**(または旧コントロールパネルの「サウンド」)
3. **「録音」タブ** → **「CABLE Output」** をダブルクリック
4. **「聴く」タブ** → **「このデバイスを聴く」にチェック**
5. 「このデバイスを使用して再生する」で **自分のヘッドホン(GSX 1000 など)** を選択 → OK

### ステップ D: 確認

味方がいる状態(カスタムやパーティ)で **味方の英語がヘッドホンから聞こえれば成功** です。
これでセットアップは完了。次回からはステップ B〜D は不要です(設定は保持されます)。

## 3. 毎回の使い方

### 起動(アプリ版・推奨)

デスクトップの **「VC Translator」** アイコン(または
`dist\VCTranslator\VCTranslator.exe`)をダブルクリックすると、GUI アプリが開きます。

1. 左上の **「▶ 翻訳を開始」** を押す(初回はモデルロードで数十秒)
2. ステータスが **「稼働中」** になったら準備完了
3. 味方が英語を話すと、ゲーム画面下の字幕 + アプリの「ライブ」タブの両方に表示されます
4. 終わったら **「■ 停止」**(履歴は自動保存されています)

Ollama(翻訳エンジン)が起動していなくても自動起動します。

### アプリのタブ構成(Editorial Ink UI)

| タブ | できること |
|---|---|
| **ライブ** | リアルタイム字幕 + 各発言への**英語返答サジェスト**。下の入力欄で**日本語→英語コール変換**。☆で気になるフレーズを保存 |
| **ライブラリ** | 試合(セッション)一覧。未レビューバッジ・発話密度バー・「今日の復習」導線 |
| **(レビュー)** | ライブラリでセッションを開くと表示。行ごとの**音声再生(0.5/0.75/1×)**・★保存・🎙シャドーイング採点・聞き逃しフィルタ |
| **復習** | ★保存したフレーズを**実際の試合音声**でフラッシュカード出題(聞き取れたら間隔が伸びる) |
| **設定** | config.yaml を GUI で編集(スライダー/トグル、即時保存) |

### 起動(コマンド版・デバッグ用)

`run.bat` からも従来どおり起動できます(引数なしで GUI が開きます)。

### 起動のバリエーション

コマンドプロンプトから使うと、用途別に切り替えられます:

```bat
cd "C:\claude code\valorant-vc-translator"

run.bat                       :: 標準(学習用・英日併記のオーバーレイ)
run.bat --profile ranked      :: ランク用(速度重視・日本語のみ)
run.bat --console             :: 字幕窓なし、黒いウィンドウに文字だけ出す
run.bat --no-translate        :: 翻訳なし(英語の文字起こしだけ確認したいとき)
run.bat --list-devices        :: 音声デバイス一覧を表示して終了
```

### Valorant を起動せずに試す

英語の練習や動作確認だけしたいときは、**YouTube の英語実況などを流し、
ブラウザ(または PC)の音声出力先を「CABLE Input」に切り替える** と、
実戦と同じ経路で字幕をテストできます。終わったら出力先を元に戻してください。

## 4. 画面表示について

- 字幕は **画面下・中央** に半透明の小窓で出ます
- **英語が薄い字で先に**、**日本語が白い太字で後から** 同じ行に入ります(二段表示)
- 古い行は自動的に消えます(既定 14 秒 / 最大 4 行)
- 字幕窓は **クリックを透過** するので、その上をクリックしてもゲーム操作の邪魔になりません

### ⚠️ フルスクリーンだと字幕が見えない

Valorant が **フルスクリーン(排他)** だと、字幕窓がゲーム画面の上に描画されません。
どちらかで対応してください:

- **Valorant 設定 → 映像 → 表示モードを「ボーダーレスウィンドウ」に変更**(おすすめ)
- または **サブモニターに字幕を出す**(最も確実)

## 5. 設定を変える(config.yaml)

`C:\claude code\valorant-vc-translator\config.yaml` をメモ帳などで開いて編集します。
編集後は run.bat を再起動すると反映されます。よく使う調整:

| 変えたいこと | 場所 | 目安 |
|---|---|---|
| 字幕を出す位置(下からの高さ) | `overlay: y_offset` | 大きくすると上に上がる |
| 字幕の横幅 | `overlay: width` | 既定 900 |
| 文字サイズ(日本語) | `overlay: font_size_ja` | 既定 17 |
| 字幕が消えるまでの秒数 | `overlay: fade_after_s` | 既定 14 |
| 半透明の濃さ | `overlay: opacity` | 0(透明)〜1(不透明) |

### プロファイルの切り替え

ファイル先頭の `profile:` を変えると、モデル構成をまとめて切り替えられます:

- `profile: learning` … 精度重視・英日併記(英語学習向け)
- `profile: ranked` … 速度重視・日本語のみ(ランク向け)

`run.bat --profile ranked` のように起動時に指定することもできます。

## 6. 用語辞書を育てる(glossary.yaml)

`C:\claude code\valorant-vc-translator\glossary.yaml` に、
エージェント名・マップ名・コール用語が登録されています。これが2つの効果を持ちます:

- **hotwords** … 音声認識のヒント(聞き取り精度が上がる)
- **terms** … 翻訳の固定対訳(誤訳を防ぐ。例: `rotate → ローテート`)

使っていて **誤認識や誤訳が出た単語** を見つけたら、ここに1行足すだけで改善します。

```yaml
terms:
  lurk: ラーク          # ← こういう行を追加していく
  "on me": 自分のところに
```

編集後は run.bat を再起動すれば反映されます。**運用しながら少しずつ足していく** のがコツです。

## 6.5. 英語学習機能の使い方

### ライブ返答提案(試合中)

翻訳の稼働中、味方の発言ごとに「こう返せる」英語例が **💬 付きでライブタブに**表示されます
(日本語の意味付き)。ゲーム画面の字幕には出ないので、プレイの邪魔になりません。
ランク用プロファイル(ranked)では負荷を減らすためオフになっています。

### 日本語→英語コール変換

ライブタブ下部の入力欄に言いたいことを日本語で入れて Enter:
「敵2人ヘブン、気をつけて」→ `Two enemies on heaven, watch out.` のような
実戦で使える英語コールに変換されます。

### ライブラリ → レビューで復習

- 試合ごとにセッションとして自動保存(英語・日本語・**実際の音声**)
- **セッションを開くと、保存済み音声を高精度で自動的に再文字起こし**(裏で実行)。
  ライブは速度優先、復習は正確なテキスト、と使い分けられる。完了した行は本文が
  更新され、単語ごとに時刻が付く
- 行をクリックすると操作列が展開:
  - **▶ 0.75×** — 本物のネイティブ発話をスロー再生(0.5/0.75/1× 切替)
  - **＋ SAVE / ★** — フレーズを保存 → 復習キューに入る
  - **🎙 SHADOW** — お手本を聞いてマネして発音 → 一致率%と誤認識語のハイライト
  - **📖 解説** — この英語コールの意味・スラング・使う場面を AI が日本語で説明
- **単語クリック再生** — 再文字起こし済みの行は英文が単語ごとに分かれ、単語をクリック
  するとその1語だけ再生。聞き取りづらい語のピンポイント練習に
- **≈ マーク+薄字** = 認識の信頼度が低い行。「聞き取りが怪しいので鵜呑みにしない」目印
- フィルタチップ「聞き逃し」= スロー再生や複数回再生した行の自動集計(苦手の見える化)
- データは `data/` フォルダ(history.db + 音声 WAV)。丸ごとバックアップ可能

### 復習(フラッシュカード)

★保存したフレーズが出題対象。**実際の試合音声**が流れるので、まず音声だけで聞き取り →
「タップして答えを表示」→ 「聞き取れた / まだ怪しい」で自己評価。
聞き取れたカードは出題間隔がどんどん伸び、怪しいカードはすぐ再出題されます
(ライブラリ右上の「今日の復習」カードから開始)。

毎日数分、復習とシャドーイングを回すだけでも「聞いて即答する」実戦力が付きます。

## 7. 困ったとき(トラブル対処)

### 字幕がまったく出ない

1. 黒いウィンドウに `=== ready: listening for speech ===` が出ているか確認
2. 味方の声が **自分のヘッドホンから聞こえているか**(聞こえない=ステップ C 未設定)
3. `run.bat --list-devices` で `CABLE Output` が見えるか確認

### 英語は出るが日本語が出ない(翻訳されない)

翻訳エンジン(Ollama)が起動していません。
スタートメニューから **「Ollama」を起動** してから run.bat を立ち上げ直してください。
ログに `Ollama に接続できません` と出ていればこれが原因です。

### 味方の声が自分に聞こえなくなった

セットアップのステップ C(「このデバイスを聴く」)が未設定か外れています。
サウンドの詳細設定 → 録音タブ → CABLE Output → 聴くタブ を再確認してください。

### 起動時に音声エラーで落ちる(PaError / stream 関連)

他のアプリが CABLE を使用中のことがあります。run.bat を一度閉じて、数秒待って再度起動してください。
(このツールは起動時に自動リトライしますが、それでも駄目な場合は再起動が確実です。)

### 認識・翻訳がおかしい / 遅い

- 誤認識・誤訳 → [用語辞書](#6-用語辞書を育てるglossaryyaml)に単語を追加
- 字幕が細切れ → `config.yaml` の `vad: min_silence_ms` を 500〜600 に上げる
- 反応が遅い → `run.bat --profile ranked` を試す

### 字幕がゲーム画面に重ならない

フルスクリーン(排他)が原因です。[セクション4](#4-画面表示について)を参照して
ボーダーレスウィンドウにするか、サブモニターに表示してください。

## 8. 終了のしかた

- 黒いウィンドウ(run.bat のウィンドウ)を **閉じる**、または `Ctrl + C` を押す
- 字幕窓も一緒に消えます

Valorant のボイスチャット出力デバイスは「CABLE Input」のままで問題ありません
(次回もそのまま使えます)。もし普通のスピーカーだけで VC を聞く状態に戻したいときは、
Valorant の設定でボイスチャットの出力デバイスを「既定のデバイス」に戻してください。

## 付録: 使う場面での注意

- **配信するとき**: 味方の英語 VC を翻訳した字幕を配信画面に映すと、第三者の声を
  無断で公開することになりかねません。字幕は自分専用モニターに留めてください。
- **チート対策(Vanguard)について**: このツールはゲーム本体に一切触れず、
  音声デバイスを読んで字幕ウィンドウを出すだけなので、検出対象になる要素はありません。
- **同時に複数人が話すと精度が落ちます**: VC は全員の声が1本にまとまって届くためです。
  戦術判断そのものより「聞き取り補助・英語学習」に向いた道具として使うのがおすすめです。
